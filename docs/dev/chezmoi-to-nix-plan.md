# Chezmoi → Nix Migration Plan

Replace chezmoi as the dotfiles materialization engine with Nix Flakes,
[home-manager](https://github.com/nix-community/home-manager), and
[nix-darwin](https://github.com/LnL7/nix-darwin). Packages move to a mix of
nixpkgs (where available and worth it) and `nix-darwin.homebrew` for casks /
formulas we still want from Brew. Secrets continue to flow through 1Password
`op://` refs read at activation time. macOS preferences move to
`nix-darwin.system.defaults` where it models the key, with custom activation
scripts (reusing `scripts/macos/plist-merge`) for nested-dict plists and
partial-merge bundles.

## Status (snapshot)

This is a **scaffolded but unvalidated** migration. The Nix code in this PR
has been written against the chezmoi sources but has not been evaluated
against `nix flake check` or built on a Mac. Expect to iterate.

Companion ADR: [docs/adr/0009-migrate-to-nix.md](../adr/0009-migrate-to-nix.md).

## Why

- Reproducibility: `darwin-rebuild switch --flake .` is a single attestable
  build; chezmoi's `apply` is a sequence of `run_onchange_*` shell scripts
  ordered by lexical filename. The latter has bitten us with subtle script
  ordering and partial-apply state.
- Atomic generations: nix-darwin generations are reversible
  (`darwin-rebuild --rollback`); chezmoi has no rollback.
- Less bespoke glue: most of `home/.chezmoitemplates/` and
  `home/.chezmoiscripts/` exists to compensate for chezmoi not modelling
  things (profile selection, license gating, plist merging at the dest, the
  cfprefsd-aware apply hooks). Several of those concerns are first-class in
  nix-darwin / home-manager.

## What changes

| Chezmoi concept                                  | Nix replacement                                                   |
| ------------------------------------------------ | ----------------------------------------------------------------- |
| `home/dot_xxx`, `home/dot_config/...`            | `home-manager` `home.file` / `xdg.configFile`                     |
| `home/.chezmoidata/packages.toml`                | `nix/modules/darwin/homebrew.nix` (data-as-nix attrset)           |
| `home/.chezmoitemplates/brewfile.tmpl`           | nix-darwin's `homebrew` module renders the Brewfile from attrs    |
| `home/.chezmoiscripts/run_once_before_*`         | `system.activationScripts.preActivation` in nix-darwin            |
| `home/.chezmoiscripts/run_onchange_after_*`     | `system.activationScripts.postUserActivation` + content-hashing   |
| `home/.chezmoitemplates/macos-defaults.sh.tmpl` | `nix-darwin.system.defaults.*` + `system.activationScripts.macos` |
| `modify_private_*.plist.tmpl` (merge stubs)     | activation script calling existing `scripts/macos/plist-merge`    |
| `modify_private_*.json|toml.tmpl` (Claude/Codex) | activation script calling existing Python mergers                 |
| `home/.chezmoi.toml.tmpl` (prompts/profile)     | per-host `nix/hosts/<host>.nix` setting `profile.*` options       |
| `home/.chezmoiexternal.toml.tmpl` (zinit clone) | home-manager `home.activation.cloneZinit` (still cloned, not pkg)|
| `home/.chezmoidata/licenses.toml`               | `profile.licenses.paths` option enumerated in nix                  |
| `home/.chezmoidata/secrets.toml`                | `profile.secrets.refs` option; resolved at activation via `op`     |
| `home/.chezmoiignore`                            | Per-module `lib.mkIf cfg.packages.<x>.enable` toggles              |
| `hooks.apply.pre` (guard running apps)           | nix-darwin activation phase: kill cfprefsd post-write              |
| `hooks.apply.post` (cfprefsd kill, relaunch)    | nix-darwin activation phase                                        |

## Layout

```text
flake.nix                       # top-level
flake.lock                      # generated by `nix flake update`
nix/
  hosts/
    prateek-mac.nix             # the personal Mac (default)
  modules/
    common.nix                  # cross-cutting profile.* options
    home/                       # home-manager modules
      default.nix
      shell.nix
      git.nix
      tools.nix
      mise.nix
      tmux.nix
      ghostty.nix
      neovim.nix
      hammerspoon.nix
      agents.nix                # claude/codex + skill rendering
      secrets.nix               # 1Password-backed file activations
    darwin/                     # nix-darwin modules
      default.nix
      homebrew.nix              # brew/cask/mas (was packages.toml + brewfile)
      defaults.nix              # NSGlobalDomain, Dock, Finder, ...
      activation.nix            # pmset, nvram, mdutil, inline plists, plist-merge
      hooks.nix                 # cfprefsd nudge, app relaunch
  lib/
    default.nix
    op.nix                      # `mkOpFile` helper for 1Password files
    plist.nix                   # `mkManagedPlist` helper for partial-merge bundles
home/                           # source files referenced by nix modules
  ...                           # files are referenced by path; chezmoi prefixes removed where needed
scripts/
  macos/                        # KEEP — plist-merge still used at activation time
  hooks/                        # renamed from chezmoi-hooks/
  audit/                        # KEEP
  trace/                        # KEEP
  vm/                           # KEEP (now invokes nix-darwin build instead of chezmoi)
  packages/                     # render-brewfile becomes a thin nix wrapper (optional)
```

The `home/` directory survives the migration as a content store — nix
modules reference its files by path. We rename leaf files where the
chezmoi prefix would be visible (`dot_zprofile` → `.zprofile`, etc.)
because home-manager doesn't strip prefixes. Top-level `home/dot_*` paths
referenced explicitly from a nix module don't need the rename; the target
name is configured in nix.

## Migration domains

### 1. Shell startup (`home/dot_zshenv.tmpl`, `home/dot_config/zsh/**`)

- `dot_zshenv.tmpl` had one template var (`{{ .dotfiles_dir }}`). Translates
  to a home-manager `programs.zsh.envExtra` snippet that uses
  `${config.home.homeDirectory}/dotfiles` (or a `profile.dotfilesDir` option).
- Everything in `home/dot_config/zsh/` is literal content. `xdg.configFile."zsh"`
  recursive-symlinks the directory. Inner `dot_zprofile`, `dot_zshrc`,
  `dot_zlogin` get renamed to `.zprofile`, `.zshrc`, `.zlogin` on disk.
- The Orca workaround in `home/dot_zshrc` stays as `home.file.".zshrc".source`.

### 2. Packages (`home/.chezmoidata/packages.toml` + brewfile.tmpl)

- Translate the TOML to a nix attrset under `nix/modules/darwin/homebrew.nix`.
- Use `nix-darwin.homebrew.{taps,brews,casks,masApps}` — nix-darwin
  generates and runs a Brewfile internally.
- Profile selection via `profile.install = "core" | "full"` option on the host
  config; the homebrew module reads the active profile's lists.
- MAS opt-in via `profile.installMas` (default false). Disabled-by-default
  matches the chezmoi `DOTFILES_INSTALL_MAS_APPS` gate.
- Xcode-required brews via `profile.installXcode` option, same shape as today.
- gh extensions via home-manager `programs.gh.extensions` (a list of
  derivations or a `home.activation.ghExtensions` script that runs
  `gh extension install` idempotently per name).
- mise tools: keep mise as the runtime/CLI manager (it already lives in
  `home/dot_config/mise/`). home-manager just materializes the config
  files; `mise install` is run via an activation script (gated on
  `profile.runInstallScripts`).

### 3. macOS defaults

Two layers:

**3a. Declarative via `nix-darwin.system.defaults.*`** — covers
~80% of `macos-defaults.sh.tmpl`:

- `NSGlobalDomain.*` (locale, animations, key repeat, scroll, …)
- `dock.*` (orientation, tilesize, autohide, hot corners)
- `finder.*` (`AppleShowAllFiles`, `ShowPathbar`, …)
- `controlcenter.*`, `WindowManager.*`, `spaces.*`
- Per-app keys via `system.defaults.CustomUserPreferences` for
  `com.apple.Safari`, `com.apple.ActivityMonitor`, `com.apple.TextEdit`,
  `com.apple.DiskUtility`, `com.apple.TimeMachine`, `com.google.Chrome`,
  `com.apple.spotlight`, `com.apple.messageshelper.MessageController`,
  `com.apple.controlcenter`, `com.apple.AppleMultitouchTrackpad`, etc.

**3b. Imperative via `system.activationScripts`** — for things nix-darwin
doesn't model:

- Inline plists: `com.apple.symbolichotkeys` (disabled hotkey IDs),
  `com.apple.HIToolbox` (keyboard layout + input sources). Emitted via
  `defaults import` from a generated plist.
- `pmset` (sleep, hibernate, standbydelay), `nvram` (SystemAudioVolume),
  `systemsetup` (timezone, restartfreeze).
- `lsregister -r` (Open With dedupe), `mdutil` (Spotlight reindex) —
  both optional, gated by env vars matching the chezmoi flags
  (`DOTFILES_SKIP_LSREGISTER`, `DOTFILES_SKIP_REINDEX`).
- `chflags nohidden ~/Library /Volumes`.
- Cfprefsd kill + optional app relaunch at the end (replaces chezmoi
  hooks/post-apply-plists.sh).

### 4. Partial-merge plists (Raycast, BTT, Tailscale, VoiceInk, Moom, …)

Keep `scripts/macos/plist-merge` (the existing uv script) unchanged. Each
bundle becomes an entry in `nix/modules/darwin/plists.nix`:

```nix
profile.managedPlists = {
  "com.raycast.macos" = ./../../home/.chezmoitemplates/com.raycast.macos.plist;
  "io.tailscale.ipn.macsys" = ./...;
  # ...
};
```

The plist templates are converted from `.plist.tmpl` (Go templates) to
plain `.plist` files where they have no Go-template logic. The Moom plist
geometry strings need careful unescaping. An activation script iterates
`profile.managedPlists` and calls `plist-merge` for each.

### 5. Secrets via 1Password

Two file-types are secret-backed:

- App licenses: `home/.chezmoidata/licenses.toml` lists target paths;
  `home/.chezmoidata/secrets.toml` carries `op://` refs.
- Per-app private templates (e.g. `home/dot_codex/modify_private_config.toml.tmpl`,
  `home/dot_claude/modify_private_settings.json.tmpl`): some of these
  embed `op://` refs in their template body.

The replacement: `nix/lib/op.nix` provides `mkOpFile { path, opRef, mode }`.
Each call adds a `home.activation` hook that runs `op read "$opRef" >
$HOME/$path` only when `profile.secrets.enabled = true` and the
1Password CLI is signed in. When disabled, the file is not written
(matching the chezmoi `secrets_enabled` toggle).

### 6. Agent skills (`home/dot_agents/packages/**`)

Keep the existing Python renderers
(`.agents/skills/agent-skill-management/scripts/render-agent-core-skills` and
`render-agent-plugin-marketplace`). They're already
chezmoi-agnostic — they just write to `$HOME/.agents/skills`,
`$HOME/.claude/skills`, `$HOME/.agents/plugins`.

- `nix/modules/home/agents.nix` exposes `profile.agents.enable` (default true).
- Activation hook (`home.activation.renderAgentSkills`) runs the renderers
  whenever the package tree hash changes. Hash is computed at evaluation
  time via `builtins.hashFile`/`builtins.hashString` over the
  `home/dot_agents/packages/` tree.
- The Claude/Codex JSON/TOML mergers
  (`home/dot_claude/modify_private_settings.json.tmpl`,
  `home/dot_codex/modify_private_config.toml.tmpl`) run as separate
  activation hooks after `renderAgentSkills`.

### 7. Hammerspoon (Fennel)

Compilation stays in the Makefile (`make hammerspoon`). nix activation
emits the compiled `init.lua` to `~/.hammerspoon/init.lua` if it exists in
the source tree, otherwise warns and falls back to the Fennel.

### 8. CI

- Drop `scripts/chezmoi/test-apply-dry-run.sh` and the chezmoi-specific
  Make targets.
- Add `nix flake check` (Linux side, fast).
- Add a macOS job that runs
  `darwin-rebuild build --flake .#prateek-mac` against the repo (no
  switch). nix-darwin has install-time prerequisites
  (`nix` installed, Determinate Systems installer or upstream
  multi-user). The Tart smoke lane installs nix in the VM before
  building.

## Rollout phases

This PR does **Phase 0**: full scaffolding, write-only. Nothing has been
run against a Mac.

- **Phase 1 (you, on a Mac)**: install nix-darwin, run
  `darwin-rebuild build --flake .#prateek-mac`. Fix the inevitable
  evaluation errors. Iterate until the build succeeds. Do _not_ `switch`
  yet.
- **Phase 2**: switch on a Tart VM, then on a personal laptop. Capture
  diffs vs. the chezmoi-rendered tree (`diff -r ~ chezmoi-output/`) and
  resolve them.
- **Phase 3**: cut `home/.chezmoi*` infrastructure entirely (already
  done in this PR, but Phase 2 may need to bring some back temporarily).
- **Phase 4**: delete the chezmoi-related parts of `docs/dev/` (now-historical
  plans like `chezmoi-migration-plan.md`, `chezmoi-agent-skills-plan.md`,
  `chezmoi-drift-banner-plan.md`) once nix is the only source.

## Open questions / TODOs

1. **Secret refs are all blank in this repo.** The mechanism is preserved
   but actual `op://` IDs live in per-machine state. The new `mkOpFile`
   helper has the same behavior: a blank ref skips the file silently.
2. **`bin/dotfiles`** was already removed before this PR; no work needed.
3. **chezmoi-drift banner** (`home/dot_config/dotfiles/chezmoi-drift/`):
   the shell startup banner that warned about uncommitted chezmoi state
   no longer makes sense. Replace with a `nix-drift` adapter that runs
   `darwin-rebuild build --dry-run` if we want similar nudging; for now
   the banner is removed.
4. **Tart smoke lane (`scripts/vm/test-install-tart.sh`)**: needs to be
   rewritten to install nix in the VM and run `darwin-rebuild build`
   instead of `chezmoi init/apply`. Not done in this PR — flagged in
   the script with a `TODO(nix)` block.
5. **Pre-commit hook** (`.pre-commit-config.yaml`): the chezmoi dry-run
   is replaced with `nix flake check`. If `nix` isn't installed locally
   the hook is a no-op (matches the chezmoi behavior).
6. **gh extensions**: home-manager's `programs.gh.extensions` accepts
   nixpkgs derivations only. For our custom extension
   (`enthus-appdev/gh-attach`), we either package it as a flake input
   or fall back to an activation script that runs `gh extension install`.
   This PR uses the activation script approach.
7. **mise auto-install**: chezmoi did `mise install` in
   `run_onchange_after_20-mise-install.sh.tmpl`. nix activation runs the
   same `mise install` if `profile.runInstallScripts = true`.
8. **The Orca `home/dot_zshrc` workaround** is preserved verbatim.

## Recovery

If the nix migration breaks your machine:

```sh
git revert <this-commit>
chezmoi apply
```

The chezmoi infrastructure deleted in this PR is recoverable from git
history. Worst case, `git checkout <pre-migration-sha> -- home/` restores
the old source tree.
