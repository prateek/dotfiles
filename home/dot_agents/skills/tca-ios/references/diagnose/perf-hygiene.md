# Applies to: TCA 1.25+, iOS 16+

# Performance Hygiene

## Use When

Use this for reducer throughput, large state copies, high action volume, or app-scale performance complaints.

## Diagnosis

- Measure before changing architecture.
- Find hot action paths.
- Count actions generated by text input, scroll, timers, and streams.
- Inspect large value copies through parent/root state.
- Check observation scope and row identity.

## Fixes

- Debounce, throttle, or batch high-frequency inputs.
- Keep large derived data out of hot state paths.
- Scope reducers and views.
- Move app-global concerns out of leaf loops.
- Consider non-exhaustive tests for huge composed flows when test brittleness is the pain.

## Pitfalls

- Do not rewrite the app architecture based on one anecdotal benchmark.
- Do not hide correctness actions just to reduce count.

## Tests

Add regression tests around debounced behavior and cancellation. Use benchmarks/profiling for throughput claims.
